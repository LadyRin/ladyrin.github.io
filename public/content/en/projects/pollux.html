Pollux

During my third year of my bachelor's program, I completed a 4-month internship at LUPM (Laboratory of Universe and Particles of Montpellier). During this internship, I worked on the Pollux software, a database of synthetic stellar spectra, accessible at https://pollux.oreme.org/.

Pollux has an admin interface, which allows managing:

  • spectra
  • collections
  • background tasks
  • shared documents (on SSH servers)
  • SSH servers
  • news articles
  • users
  • roles
However, this interface was developed in Vue.js 2, which is now obsolete. My task was to migrate this interface to Vue.js 3, while adding additional features.

During the project, I developed reusable components to reduce the code duplication present in the original version. I also implemented an end-to-end testing solution using Cypress, which allowed me to verify the correct functioning of the application. Thanks to these tests, I ensured that the application met expectations in terms of functionality and interaction with the API. This improved the maintainability and reliability of the code.

Regarding the management of communication with the API, I noticed a lot of redundancy in the dedicated functions. To solve this problem, I created a generic class for API services, which made managing HTTP requests (GET, POST, PUT, DELETE) more efficient. This made the application more modular and easier to maintain in the long term.

Class diagram of generic services

Finally, I made improvements to the authentication and flash message management system. The old system was not very secure, so I opted for a token-based authentication system using JWT. I also designed a new flash message management system to better inform users of successful actions or errors within the application.
