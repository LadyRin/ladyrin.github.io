E:cclesia

E:cclesia est un site web de décision démocratique. Il permet à ses utilisateurs de créer des questions ouvertes, de les soumettre à la communauté et de voter pour les réponses proposées.

Cycle de vie d'une question

À tout moment, un utilisateur peut créer une demande de question. Il y définit le sujet et une courte description. Une fois la question validée par un administrateur, une description plus poussée, un calendrier de vote, ainsi que le système de vote qui sera utilisé sont ajoutés. La question est ensuite publiée.

Les utilisateurs peuvent alors proposer des réponses à la question, suivant le plan de réponse défini par l'auteur. Une fois la phase de rédaction terminée, les utilisateurs peuvent voter pour les réponses qu'ils préfèrent.

Une fois la phase de vote terminée, les résultats sont affichés.

Systèmes de vote

E:cclesia propose plusieurs systèmes de vote, chacun adapté à un type de question.

  • Scrutin uninominal à un tour : chaque utilisateur peut voter pour une seule réponse.
  • Vote par approbation : chaque utilisateur peut voter pour plusieurs réponses.
  • Vote alternatif : chaque utilisateur classe les réponses dans l'ordre de ses préférences. Les réponses sont ensuite classées selon l'algorithme du vote alternatif
  • Jugement majoritaire : chaque utilisateur attribue une note à chaque réponse (de "À rejeter" à "Très bien") et les réponses sont ensuite classées selon l'algorithme du jugement majoritaire

Réalisation

E:cclesia est un projet réalisé en 4 mois par 4 étudiants de l'Université de Montpellier. Il a été réalisé en utilisant les langages et technologies suivants :

  • HTML
  • SCSS
  • JavaScript
  • PHP
  • PostgreSQL
  • Git

Le projet à été réalisé dans le cadre d'une SAÉ (Situation d'Apprentissage et d'Évaluation) de 2ème année de BUT Informatique. Nous avons notamment utilisé la méthode agile SCRUM pour la gestion de projet.
